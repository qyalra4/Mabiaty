# دليل تجهيز ورفع تحديث مبيعاتي على GitHub

هذا المجلد خاص بتجربة ميزة التحديث التلقائي داخل تطبيق سطح المكتب `مبيعاتي`.

## الملفات الموجودة

- `Mabiaty_Update_v1.0.1.zip`
  ملف التحديث الذي ترفعه إلى GitHub كـ Release Asset.

- `RELEASE_NOTES_v1.0.1.md`
  ملاحظات الإصدار التي يتم وضعها داخل الحزمة باسم `read.md` حتى تظهر للمستخدم بعد التحديث.

- `GITHUB_RELEASE_BODY.md`
  نص جاهز تضعه داخل وصف GitHub Release.

- `package-update.ps1`
  سكربت يعيد إنشاء ملف ZIP من ملفات البناء الإنتاجية.

## ما يحتويه ملف ZIP

ملف ZIP يحتوي ملفات تشغيل التطبيق فقط:

- `SalesManager.exe`
- ملفات Qt DLL المطلوبة
- مجلدات Qt runtime مثل `platforms`, `sqldrivers`, `imageformats`
- مجلد `assets`
- مجلد `styles`
- ملف `read.md` الخاص بملاحظات التحديث

## ما لا يحتويه ملف ZIP

حزمة التحديث لا تحتوي بيانات المستخدم:

- لا تحتوي `data/app.db`
- لا تحتوي قواعد بيانات المتاجر
- لا تحتوي ملفات PDF التي تم إنشاؤها
- لا تحتوي ملفات CMake أو ملفات البناء الداخلية

هذا مهم حتى لا يمسح التحديث بيانات العميل.

## طريقة إنشاء Release على GitHub

1. افتح مستودع GitHub الخاص بالمشروع.
2. ادخل إلى صفحة `Releases`.
3. اضغط `Draft a new release`.
4. اكتب Tag مثل:

   ```text
   v1.0.1
   ```

5. اكتب عنوان الإصدار:

   ```text
   مبيعاتي v1.0.1
   ```

6. انسخ محتوى `GITHUB_RELEASE_BODY.md` وضعه في وصف الإصدار.
7. في قسم Assets اضغط رفع ملف.
8. ارفع الملف:

   ```text
   Mabiaty_Update_v1.0.1.zip
   ```

9. اضغط `Publish release`.

## طريقة التجربة داخل التطبيق

1. شغل تطبيق `مبيعاتي`.
2. افتح شاشة `الأدوات`.
3. افتح تبويب `التحديثات`.
4. في حقل رابط التحديث ضع رابط المستودع:

   ```text
   https://github.com/USER/REPO
   ```

   استبدل `USER` باسم حسابك و `REPO` باسم المستودع.

5. اضغط `حفظ الرابط`.
6. اضغط `فحص التحديث`.
7. إذا وجد التطبيق Release يحتوي ملف ZIP، سيظهر زر التحميل.
8. اضغط `تحميل وتثبيت التحديث`.
9. التطبيق يغلق نفسه، يستبدل الملفات، ثم يفتحه مرة ثانية.
10. عند الفتح بعد التحديث تظهر نافذة معلومات التحديث من ملف `read.md`.

## رابط ZIP مباشر

يمكن أيضاً وضع رابط مباشر لملف ZIP بدل رابط المستودع:

```text
https://github.com/USER/REPO/releases/download/v1.0.1/Mabiaty_Update_v1.0.1.zip
```

بهذه الحالة التطبيق يتجاوز قراءة GitHub API ويبدأ من الرابط المباشر.

## ملاحظات مهمة للإنتاج

- لا ترفع قواعد بيانات العملاء داخل التحديث.
- لا تغير اسم الملف التنفيذي الرئيسي داخل ZIP.
- خلي ملفات ZIP تحتوي الملفات مباشرة في الجذر، وليس داخل مجلد داخلي.
- إذا كان التطبيق مثبت داخل `Program Files` قد يحتاج صلاحيات مدير حتى يستبدل الملفات.
- للتوزيع النهائي الأفضل توقيع `SalesManager.exe` وملف ZIP بتوقيع رقمي.
- ميزة التحديث الحالية مناسبة للتجربة. للإنتاج الكامل يفضل إضافة فحص SHA256 أو توقيع للملف قبل التثبيت.

## إعادة إنشاء الحزمة

من PowerShell داخل مجلد `SalesManager` شغل:

```powershell
.\release-updates\package-update.ps1
```

سيتم إنشاء:

```text
release-updates\Mabiaty_Update_v1.0.1.zip
```
